## Vercel

aixyz has first-class Vercel support using the [Build Output API v3](https://vercel.com/docs/build-output-api/v3).

```bash
bun run build
vercel deploy
```

The build automatically detects the Vercel environment via `VERCEL=1` and outputs a Vercel serverless function using Bun Runtime. The agent URL is auto-detected from Vercel environment variables.

### `vercel.json`

Every aixyz project includes a `vercel.json`:

```json title="vercel.json"
{
  "$schema": "https://openapi.vercel.sh/vercel.json",
  "framework": null,
  "buildCommand": "bun run build",
  "bunVersion": "1.x"
}
```

No need to pass `--vercel` — the flag is automatically detected during Vercel builds.

Set `OPENAI_API_KEY` and any payment environment variables in the Vercel dashboard under **Settings → Environment Variables**.

## Standalone (Bun Runtime)

For direct deployment on Bun-compatible hosts (DigitalOcean, Fly.io, etc.):

```bash
aixyz build
bun .aixyz/output/server.js
```

The build outputs a single bundled file at `.aixyz/output/server.js`.

### Environment Variables

Set these before running:

```bash
export PORT=3000
export OPENAI_API_KEY=sk-...
bun .aixyz/output/server.js
```

## Railway

[Railway](https://railway.com) auto-detects a `Dockerfile` in your project root. Use a multi-stage build so only the compiled output is shipped in the final image, and pass `RAILWAY_PUBLIC_DOMAIN` at build time so the agent URL is resolved automatically.

```dockerfile title="Dockerfile"
FROM oven/bun:1 AS build
WORKDIR /app
COPY . .
RUN bun install
ARG RAILWAY_PUBLIC_DOMAIN
ENV RAILWAY_PUBLIC_DOMAIN=$RAILWAY_PUBLIC_DOMAIN
RUN bun run build

FROM oven/bun:1-slim
WORKDIR /app
COPY --from=build /app/.aixyz/output .aixyz/output
CMD ["bun", ".aixyz/output/server.js"]
```

Deploy by connecting your GitHub repository to a Railway project. Railway injects `RAILWAY_PUBLIC_DOMAIN` automatically so `bun run build` picks up the correct public URL — no manual `url` config needed.

Set `OPENAI_API_KEY` and any payment environment variables in the Railway dashboard under **Variables**.

Add a `.dockerignore` to keep the build context small:

```text title=".dockerignore"
node_modules
.aixyz
.vercel
.env*.local
*.log
```

<Note>
  `RAILWAY_PUBLIC_DOMAIN` is only available after a public domain is assigned in Railway. If you use a custom domain,
  set it there first and Railway will expose it as the `RAILWAY_PUBLIC_DOMAIN` build argument.
</Note>

## Docker

For self-hosted or custom container deployments, use a multi-stage build to keep the final image small:

```dockerfile title="Dockerfile"
FROM oven/bun:1 AS build
WORKDIR /app
COPY . .
RUN bun install
RUN bun run build

FROM oven/bun:1-slim
WORKDIR /app
COPY --from=build /app/.aixyz/output .aixyz/output
CMD ["bun", ".aixyz/output/server.js"]
```

Build and run:

```bash
docker build -t my-agent .
docker run -p 3000:3000 \
  -e OPENAI_API_KEY=sk-... \
  -e PORT=3000 \
  my-agent
```

Add a `.dockerignore` to keep the build context small:

```text title=".dockerignore"
node_modules
.aixyz
.vercel
.env*.local
*.log
```

<Warning>Never bake API keys into Docker images. Pass them as environment variables at runtime.</Warning>

<Note>
  Set the `url` field in `aixyz.config.ts` manually for Docker deployments since Vercel environment variables won't be
  available.
</Note>

## Build Output Formats

| Format     | Flag                            | Output Path               | Description                                                                       |
| ---------- | ------------------------------- | ------------------------- | --------------------------------------------------------------------------------- |
| Standalone | (default)                       | `.aixyz/output/server.js` | Single-file bundle, run with `bun`                                                |
| Vercel     | `VERCEL=1` or `--output vercel` | `.vercel/output/`         | [Build Output API v3](https://vercel.com/docs/build-output-api/v3) for serverless |
| Executable | `--output executable`           | `.aixyz/output/server`    | Self-contained binary, no Bun runtime required                                    |

## Verifying Your Deployment

After deploying, verify your agent is accessible:

```bash
# Check the agent card
curl https://your-agent.example.com/.well-known/agent-card.json

# Test the A2A endpoint
curl -X POST https://your-agent.example.com/agent \
  -H "Content-Type: application/json" \
  -d '{
    "jsonrpc": "2.0",
    "method": "tasks/send",
    "id": "1",
    "params": {
      "id": "task-1",
      "message": {
        "role": "user",
        "parts": [{"type": "text", "text": "Hello"}]
      }
    }
  }'
```
